Exercise 2 - E-commerce Platform Search Function
=================================================

Objective:
Implement and compare linear and binary search for finding products by name.

Big O Notation:
Big O describes how an algorithm's runtime grows as input size (n) increases.
It helps predict performance and choose the right algorithm at scale.

Search Scenarios:
- Best case:    The item is found immediately (first element or midpoint)
- Average case: The item is somewhere in the middle of the list
- Worst case:   The item is at the end or does not exist

Linear Search:
- Best:    O(1) - found at index 0
- Average: O(n/2) simplified to O(n)
- Worst:   O(n) - scanned entire array

Binary Search:
- Best:    O(1) - found at midpoint on first try
- Average: O(log n)
- Worst:   O(log n) - requires a sorted array

Which to use?
For a small or unsorted catalog, linear search is simpler and works fine.
For large sorted product catalogs, binary search is far more efficient.
Searching 1 million products: linear takes up to 1,000,000 steps, binary takes ~20.
