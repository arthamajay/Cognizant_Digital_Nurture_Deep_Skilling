Exercise 4 - Employee Management System
========================================

Objective:
Use an array to manage employee records with add, search, traverse, and delete operations.

Array Memory Representation:
Arrays store elements in contiguous memory blocks. Each element is accessed via an index,
which maps directly to a memory address (base address + index * element size).
This makes index-based access extremely fast.

Advantages of Arrays:
- O(1) access by index
- Simple structure, low memory overhead
- Cache-friendly due to contiguous storage

Time Complexity Analysis:
- Add:      O(1) - insert at next open index (when array is not full)
- Search:   O(n) - must scan each element to find a match by ID
- Traverse: O(n) - visits every element once
- Delete:   O(n) - requires finding the element and shifting remaining elements left

Limitations of Arrays:
- Fixed size: must declare capacity upfront
- Deletion is costly due to element shifting
- No dynamic resizing without creating a new array

When to use Arrays:
Arrays are ideal when the dataset is small and fixed in size, when fast index-based
access is needed, or when memory efficiency is a priority. For dynamic employee records
that grow and shrink frequently, an ArrayList or LinkedList would be more appropriate.
